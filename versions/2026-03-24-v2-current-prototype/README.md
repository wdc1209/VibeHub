# Vibe Hub for OpenClaw

一个可立刻打开手动测试的 **Vibe Hub 当前稿前端原型**。

这版继续沿着 companion app 方向推进，但重点变成两件事：
- 让主卡片可以在桌面上拖动，用来观察不同背景位置下的毛玻璃/漂浮效果
- 在同一套内容编辑与发送交互上，切换比较 3 种明显不同的非圆角矩形外形

---

## 这次新增了什么

### 1) 主卡片可拖动
- 支持鼠标拖动
- 拖动区域在卡片顶部
- 会尽量限制在当前视口范围内，方便测试不同桌面位置的观感
- 拖动后位置会保存在本地 `localStorage`

### 2) 三种外形可切换
顶部新增轻量切换区，可以比较这 3 个版本：
- **透明信封形**
- **透明对话气泡形**
- **切角 Token 牌形**

这三种不是简单改圆角，而是整体轮廓与高光语言都不同，适合快速做风格判断。

### 3) 重写 / 压缩反馈更明显
- 点击 `重写` / `压缩` 后，除了 toast 之外，正文区标题右侧也会出现短暂反馈 badge
- 更容易一眼看到这次操作到底有没有生效

### 4) 背景更像桌面 / 壁纸
- 保留玻璃浮层感
- 加了更接近真实桌面的壁纸层、装饰块和底部 dock 氛围
- 不是单纯纯色演示底

## 现在能测试什么

### A. 外形比较
你可以在顶部按钮组里切换：
- 透明信封
- 透明对话气泡
- 切角 Token 牌

然后直接比较：
- 哪一种更像系统浮层
- 哪一种更适合 Vibe Hub 这个名字
- 哪一种在毛玻璃下最稳定
- 哪一种最不像普通圆角面板

### B. 不同桌面位置下的毛玻璃效果
- 抓住卡片顶部拖动
- 把卡片拖到左上 / 中间 / 靠右 / 靠近底部 dock 一带
- 观察不同背景区域下的折射、亮度和漂浮感

### C. 现有交互仍然可用
以下能力继续保留：
- 编辑正文
- 切发送目标
- `发送 / 重写 / 压缩 / 取消`
- `Token Stack` 历史抽屉
- 历史卡查看详情
- 再次发送 / 编辑后发送 / 转发给其他目标

### D. 重写 / 压缩是否足够显眼
现在可以测试：
- toast 提示是否够明显
- 正文区右上角反馈 badge 是否更直观
- `已重写` / `已压缩` 状态是否足够帮助用户确认系统做了事

## 当前界面方向

仍然保持前面确认过的简洁方向：
- 左上角只保留 `Vibe Hub`
- 发送目标放在标题右侧附近
- 不做大块表单感
- 页面整体尽量干净，突出主卡片本身

---

## 代码里的 OpenClaw 接口位

当前 `app.js` 里仍保留 mock bridge：

- `bridge.createVibeHub(payload)`
- `bridge.sendVibeHub(payload)`
- `bridge.rewriteToken(payload)`
- `bridge.shortenToken(payload)`

现在这些还是前端 mock，但结构上已经能替换为真实接线。

后续可以接到例如：
- OpenClaw 本地 bridge
- `opencli codex send`
- companion app 的原生壳层 IPC
- 本地数据库 / 文件存储 / agent dispatch service

---

## 这稿更适合做什么测试

最适合拿来测 3 件事：
1. **拖动后的桌面观感**：卡片在不同背景位置是否仍然高级
2. **外形方向判断**：3 种版本里哪一个最像最终产品
3. **轻交互是否顺手**：切目标、改正文、重写、压缩、发送是否仍然足够轻

## 我个人建议先看哪一种

如果是这一稿第一次打开，我建议先看：

**切角 Token 牌形**

原因很简单：
- 它最不像普通玻璃卡片
- 跟 `Vibe Hub` 这个名字最贴
- 作为产品识别度会更强
- 同时还保留了足够干净的编辑区域，不会太花

如果想要更温和、更接近系统浮层，再回头看 **透明信封形**；
如果想测试更强的“发给某个 agent 的消息载体感”，可以看 **透明对话气泡形**。
