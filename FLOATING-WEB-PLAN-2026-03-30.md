# Vibe Hub Web Floating Plan — 2026-03-30

## 1. 目标

当前优先级不是继续接新目标软件，而是先把 `Vibe Hub` 的“漂浮”能力重新定义清楚，并给网页端补上一套可调参数。

这次的目标不是做夸张动画，而是做一个：

- 可开关
- 可调强弱
- 不影响可读性
- 不引起“下沉 / 溺水 / 文本和壳不同步”感受
- 未来能和原生版共享参数语义

一句话定义：

> 漂浮不是特效，而是 Vibe Hub 作为前台工作台的存在感参数。

## 2. 为什么之前会反复回退

从当前项目记录看，之前漂浮被多次回退，根因已经很明确：

1. 把漂浮做在内容层，而不是整体卡片层
2. 位移幅度过大，用户会感觉窗口在“下沉”
3. 漂浮和阴影 / 毛玻璃 / 文本清晰度没有一起设计
4. 原生端和网页端没有共用同一套参数语言，导致每次都像在重新试错

这也是为什么这次不能只说“恢复漂浮”，而是要先定义参数模型。

## 3. 本轮设计原则

### P1. 漂浮默认应非常轻

默认状态应该更接近：

- 轻微呼吸
- 微小悬停
- 稳定前台卡片

而不是：

- 明显上下飘动
- 浮标感
- 摇晃感

### P2. 漂浮只作用于整张卡

无论网页端还是原生端，漂浮都应理解为：

- 卡片外壳
- 阴影
- 光晕
- 内容

一起动，不能出现内容像固定住、背景壳在移动的错位感。

### P3. 漂浮应低优先于可读性

如果漂浮参数和以下目标冲突，优先保留这些：

- 文本可读
- 编辑稳定
- 按钮可点
- 焦点不跳

### P4. 网页端先做“参数化”，再做“自动化”

本轮网页端重点不是追求最复杂的动态，而是先让漂浮有一组明确参数，并可在网页端直接控制。

## 4. 网页端应新增的漂浮参数

网页端建议新增一个 `floating` 配置对象。

建议字段如下：

```js
floating: {
  enabled: true,
  mode: "subtle",
  amplitudeY: 4,
  amplitudeX: 0,
  scaleBreath: 0.006,
  durationMs: 6200,
  glowStrength: 0.18,
  shadowStrength: 0.22,
  blurBoost: 0,
  tiltDeg: 0,
  dragPreservesPosition: true,
  pauseOnInputFocus: true,
  pauseOnHover: false,
  reduceMotionRespect: true
}
```

## 5. 每个参数的含义

### `enabled`

是否启用漂浮。

### `mode`

建议只保留 3 档：

- `off`
- `subtle`
- `present`

不要一开始做成过细的 10 档滑杆，否则会让体验判断变得模糊。

### `amplitudeY`

垂直漂浮位移，单位 `px`。

建议：

- 默认：`3~4`
- 上限：`8`

超过这个范围，Vibe Hub 很容易重新出现“下沉感”。

### `amplitudeX`

水平位移。

建议默认保持 `0`。网页端第一阶段不建议加横向漂移。

### `scaleBreath`

整体呼吸缩放幅度。

建议范围：

- `0`
- `0.004`
- `0.006`

不要超过 `0.01`。

### `durationMs`

完整一轮漂浮周期。

建议默认：

- `5800~7200ms`

过快会像抖动，过慢则用户感知不到。

### `glowStrength`

控制边缘光晕存在感。

### `shadowStrength`

控制卡片漂浮感的支撑阴影强度。

这里很关键：

- 阴影太轻，卡片像贴在背景上
- 阴影太重，卡片像压下去

### `blurBoost`

为毛玻璃提供一个小的增强量。

建议初期默认 `0`，只有在背景太杂时才手动加一点。

### `tiltDeg`

倾斜角。

建议第一阶段固定 `0`，先不做。

### `dragPreservesPosition`

网页端已有拖动位置逻辑，漂浮必须和手动拖拽兼容。

规则应为：

- 用户拖到哪里，卡片就以那个点为“漂浮锚点”
- 漂浮围绕锚点轻微摆动，不可重新把卡片拉回中心

### `pauseOnInputFocus`

当正文编辑框或原始输入框处于激活状态时，暂停漂浮。

这是非常值得保留的参数，因为编辑时用户对稳定性更敏感。

### `pauseOnHover`

可选。

我建议默认 `false`，因为 hover 停止会让动画语义过碎。

### `reduceMotionRespect`

如果用户系统偏好 `prefers-reduced-motion`，则自动退到：

- 只保留轻阴影变化
- 或完全关闭漂浮

## 6. 网页端参数面板建议

网页端不需要一次暴露所有底层参数给普通用户。

建议分两层：

### 普通设置

- 漂浮：开 / 关
- 漂浮强度：轻 / 中
- 编辑时暂停漂浮：开 / 关

### 高级设置

- 垂直位移
- 呼吸缩放
- 周期
- 光晕强度
- 阴影强度

这样既保留了可调性，也不会让网页端变成动画实验台。

## 7. 实现建议

### 第一阶段：网页原型参数化

在网页原型中先做这几件事：

1. 为现有 `floatingPosition` 增加独立的 `floating` 配置
2. 漂浮动画围绕当前拖拽位置锚点计算
3. 焦点进入 `整理后内容` / `原始输入` 时暂停漂浮
4. 增加网页端参数面板
5. 默认模式设为 `subtle`

这一阶段的目标是：

> 在网页端稳定找到“看起来活着，但不烦人”的默认参数。

### 第二阶段：参数映射到原生版

等网页端参数稳定后，再映射到原生版。

原生端不必一开始全量跟进，但要保持字段语义一致，例如：

- `enabled`
- `mode`
- `amplitudeY`
- `durationMs`
- `pauseOnInputFocus`

### 第三阶段：保存与同步

后续如果网页端和原生端都要保留这个能力，参数应写入统一配置层，而不是散在样式和代码里。

## 8. 默认值建议

建议网页端第一版默认值如下：

```js
floating: {
  enabled: true,
  mode: "subtle",
  amplitudeY: 4,
  amplitudeX: 0,
  scaleBreath: 0.005,
  durationMs: 6400,
  glowStrength: 0.16,
  shadowStrength: 0.20,
  blurBoost: 0,
  tiltDeg: 0,
  dragPreservesPosition: true,
  pauseOnInputFocus: true,
  pauseOnHover: false,
  reduceMotionRespect: true
}
```

## 9. 不做的事

本轮明确不做：

- 大幅上下飘动
- 左右漂移
- 卡片倾斜旋转
- 文本层和背景层分离运动
- 为每个小面板单独做漂浮
- 自动吸附屏幕边缘

这些要么会影响稳定性，要么会偏离 Vibe Hub 的工作台定位。

## 10. 验收标准

网页端漂浮方案完成后，应至少满足：

1. 默认状态下用户能感知到轻微生命感
2. 但不会再出现“下沉”“溺水”“晃动”反馈
3. 拖动后仍围绕新位置稳定漂浮
4. 文本输入时漂浮暂停或足够轻，不影响编辑
5. 参数可在网页端显式调整
6. 不开启高级设置也能得到合理默认效果

## 11. 下一步执行顺序

建议按这个顺序推进：

1. 先在网页原型里补 `floating` 参数对象
2. 再补网页设置面板
3. 再把编辑态暂停逻辑接上
4. 最后做一轮默认值校准

当前结论：

> 漂浮这条线不该再靠“感觉回调”，而应先用网页端参数化收敛，再把稳定参数带回原生版。
